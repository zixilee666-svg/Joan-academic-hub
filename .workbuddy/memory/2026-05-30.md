## 06. 修复 parse-paper AI_PARSE_ERROR — 双重保险排除 /api/ai/* (commit: 18fea44)

根因: `pages.config.json` 中的负向先行断言 `(?!ai/)` 在 EdgeOne Pages 的正则引擎中不生效

2. edge-functions: chat/parse-paper handler 返回 503 MIGRATED_TO_CF

3. Cloud Function 保持不变: Node.js v20, 120s 超时, AI API 正常调用

## 07. 彻底修复 AI 路由 — 移除 Edge Function catch-all (commit: 82fc74a)

### 问题
修复 #05/#06 后 parse-paper 仍返回 503 MIGRATED_TO_CF
真正根因: `[[default]].js` 在 EdgeOne Pages 中是 **catch-all** 文件名
→ 无论 pages.config.json regex 如何排除，它都拦截所有 /api/* 请求
→ Cloud Function 永远无法被调用

### 修复 (三管齐下)
1. `edge-functions/api/[[default]].js` → `_kv-handler.js`
   → 普通文件名，不再是 catch-all
2. `pages.config.json`: `^/api/(?!ai/)(.*)$` → `_kv-handler.js`
   → 非 catch-all 文件 + regex 排除 = 真正生效
3. Cloud Function `cloud-functions/api/ai/[[default]].js` 维持不变
   → Node.js v20 + 120s超时 + 外部网络 API 调用

### 路由结果
| /api/auth/*      | regex匹配 → _kv-handler.js (Edge Function, KV存储) |
| /api/papers/*    | regex匹配 → _kv-handler.js (Edge Function) |
| /api/ai/*        | regex不匹配 → Cloud Function (AI API调用) |
