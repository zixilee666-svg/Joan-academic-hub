## 知识图谱边标签修复

**问题**: 知识图谱可视化中所有边的关系标签显示为 "undefined"。

**根因**: 数据层 (`data/knowledgeGraph.ts`) 使用字段名 `type` 存储关系类型（如 `{ source, target, type: 'is_a' }`），但可视化组件 `KnowledgeGraphCanvas.tsx` 的 props 定义和映射逻辑读取的是字段名 `relation`（`l.relation`）。字段名不匹配导致 `relation` 始终为 `undefined`。

**修复内容**:
1. `KnowledgeGraphCanvas.tsx` props 的 `links` 类型同时支持 `relation?: string` 和 `type?: string`
2. links useMemo 映射逻辑改为 `const relType = l.relation || l.type || ''`，优先取 `relation`，回退到 `type`
3. `RELATION_LABELS` 补充缺失的 `uses: '使用'` 映射

**涉及文件**:
- `src/components/knowledge/KnowledgeGraphCanvas.tsx`

**验证**: TypeScript `--noEmit` 零错误通过。
**GitHub**: commit `7636d69`，已推送到 `origin/main`。

## 知识图谱布局稳定性修复

**问题**: 子图可视化中节点数量不足（7实体仅显示3-4个），节点重叠/飞出视口，边标签仍可能显示英文。

**根因**:
1. 初始布局使用纯随机坐标，力导向模拟收敛不稳定
2. 排斥力常数过大（1800²），小图（节点少、边少）中节点被弹飞
3. 中心引力过弱（0.0008），无法约束节点在可视区域内

**修复内容**:
1. **环形初始布局** — 按角度均匀分布节点，取代随机初始化
2. **自适应物理参数** — 根据节点数动态调整排斥力（≤5节点用800²，>5用1400²）、弹簧强度（稀疏图0.015）、中心引力（≤5节点用0.003）
3. **安全距离下限** — 排斥力计算加入 `safeDist = max(dist, a.radius + b.radius + 10)`，避免节点过近时力爆炸

**涉及文件**:
- `src/components/knowledge/KnowledgeGraphCanvas.tsx`

**验证**: TypeScript + Vite build 零错误。
**GitHub**: commit `8df07ea`，已推送到 `origin/main`。

## DOCX 资料预览修复

**问题**: 导入的 `.docx` 文件在资料查看页面显示「此资料暂无可预览内容」。

**根因**:
1. `ImportExportPage.tsx` 导入 docx 时，仅对 `txt/md` 提取文本内容，`docx` 的 `content` 字段为空字符串
2. `fileUrl` 字段未传递（项目无文件存储，仅 KV），后端存为空字符串
3. `MaterialViewPage.tsx` 第224行判断 `if (!url)` → url 为空 → 直接返回「无可预览内容」提示

**修复内容**:
1. **`ImportExportPage.tsx`** — docx 导入分支增加 `extractTextFromFile(file)` 调用，将提取的纯文本存入 `content` 字段
2. **`MaterialViewPage.tsx`** — docx 预览逻辑增加回退：优先用 `fileUrl`（远程文件），若为空则使用 `material.content`（KV 存储的文本），渲染为 `<TextViewer>`

**涉及文件**:
- `src/pages/ImportExportPage.tsx`
- `src/pages/MaterialViewPage.tsx`

**验证**: TypeScript + Vite build 零错误。
**GitHub**: commit `4e84789`，已推送到 `origin/main`。

## 所有格式预览修复（PDF/DOCX/MD/TXT）

**问题**: 用户反馈「pdf、docx、md、txt格式均预览不了」。

**根因**: `MaterialViewPage.tsx` 的 `renderViewer` 函数中，`if (!url)` 判断（第224行）在所有内容检查之前就返回了「无可预览内容」，导致无 `fileUrl` 的资料（当前项目无文件存储服务，仅 KV）全部无法预览。

**修复内容**:

### 1. 重写 `renderViewer` 预览优先级（`MaterialViewPage.tsx`）

```
优先级1（content）：TXT/MD/DOCX → TextViewer（KV存储文本）
优先级2（fileUrl）：PDF/DOCX → PdfViewer/DocxViewer（需远端文件）
优先级3（回退）：均无 → 提示无可预览 + 下载按钮
```

关键改动：`if (!url)` 判断被移除，改为**先检查 content，再检查 fileUrl**。

### 2. 导入时 PDF 也提取文本（`ImportExportPage.tsx`）

原逻辑只对 `txt/md` 提取文本，`pdf/docx` 的 `content` 为空。现改为：
- `txt/md` → `file.text()`（原逻辑）
- `docx/pdf` → `extractTextFromFile(file)`（新增）

**涉及文件**:
- `src/pages/MaterialViewPage.tsx`
- `src/pages/ImportExportPage.tsx`

**验证**: TypeScript + Vite build 零错误。
**GitHub**: commit `fed7dd1`，已推送到 `origin/main`。
