## 26. 学术搜索修复 - Edge Function 路由拦截根因 (commit: 13677ea)
- **现象**：截图显示搜索返回 `[v1]` / `"version":"ai-entry"`，即 Edge Function 的响应

## 14. 修复 hover 抖动排斥 + 节点点击无知识图谱 (commit: f6f5e50)

### 问题 1: 节点点击无知识图谱
- **根因**: `NODE_LINKS` 映射表路径为 `/knowledge?topic=XXX`，但 App.tsx 中实际路由定义在 `/dashboard/knowledge`（嵌套于 `/dashboard` 父路由下，HashRouter）
- **修复**: 所有 16 条 NODE_LINKS 路径从 `/knowledge?topic=XXX` → `/dashboard/knowledge?topic=XXX`，fallback 同步修正

### 问题 2: Hover 抖动排斥
- **根因**: CSS `.node-visual { transition: transform 0.25s ease }` + hover 时 `transform: scale(1.15)` 与 JS raf 动画循环的 `setAttribute('transform', 'translate(...)')` 产生竞态冲突
  - 父 `<g>` 被 JS 持续移动 → hover 时子元素 CSS scale 改变视觉边界 → 鼠标可能"离开"节点 → hover 状态切换 → scale 恢复 → 鼠标又"进入" → 快速循环抖动
  - 周围节点看似被"排斥"是因为缩放节点改变了与相邻节点的相对位置感知
- **修复方案**: 完全移除 `transform: scale(1.15)`，改用纯 filter 实现 hover 强调：
  - 移除 `.node-visual { transform-origin: center; transition: transform ... }` 
  - hover 效果改为 `filter: brightness(1.35) drop-shadow(0 0 16px rgba(255,215,0,0.65))`
- **经验**: SVG 中「JS 控制位置的 transform」与「CSS hover 的 transform scale」是死敌，必须二选一。hover 反馈应优先用 filter/stroke-width/opacity 等非 transform 属性

## 7. JoanLearningGNN Hover节点跳动修复 (commit: 64ae47d)

**现象**: 鼠标移到知识节点上时，节点会排斥/跳动
**根因**: CSS `transform: scale(1.15)` 覆盖了 JS raf 每帧设置的 `transform: translate(cx,cy)`
       → 节点位置被重置到原点附近 → JS下一帧又弹回轨道 → 快速抖动
**修复**: 双层 `<g>` 分离架构
  - 外层 `<g data-orbit-node>`: 仅负责位置 translate（JS raf 驱动）
  - 内层 `<g className="node-visual">`: 负责所有视觉内容 + hover 缩放
  - CSS `.knowledge-node-group:hover .node-visual { transform: scale(1.15) }` 只影响内层

### 经验教训
- **SVG动画 + CSS transform 绝对不能混用在同一元素上**
- JS raf 控制位置时，hover 视觉效果必须放在子元素上
- 这是 SVG 开发中的经典坑，以后做任何 SVG 交互都要注意

## 23. 修复搜索仍然返回空数组 — 路由冲突根因 (commit: 926c895)

## 27. 学术搜索 Cloud Function v4-cached (commit: b2b738c)
- **背景**：v3-retry 路由修复后，搜索到达 Cloud Function，但外部 API 有障碍：
  - Semantic Scholar: 429 Too Many Requests（限流）
  - arXiv: 超时（国内访问极慢）
- **v4 改进**：
  1. **内存缓存层**：Map + TTL（arXiv 24h, S2 1h），同 query 避免重复请求外部 API，缓存命中返回 `_cached: true`
  2. **arXiv 超时 45s**：`fetchWithRetry` 可配置 timeout，arXiv 默认 45s（vs v3 的 25s）
  3. **Semantic Scholar 4次重试**：429 退避 max 32s（vs v3 的 2次/16s）
  4. **缓存清理**：每5分钟 `setInterval` 清理过期条目
- **文件**：
  - `.edgeone/cloud-functions/api-search/index.mjs` (tracked in git despite .gitignore)
  - `cloud-functions/api-external/search/[[default]].js` (source)
  - `pages.config.json` (Edge Function route fix from 13677ea)
- **已推送 → EdgeOne Pages 自动部署**

### 核心经验 — EdgeOne Pages 双路由机制
- **Edge Function** (`pages.config.json` → `edgeFunctions.routes[].src`)：路径模式匹配（不含 query string），`^/api/([a-z]+)(/.*)?$` 只匹配纯字母路径
- **Cloud Function** (`.edgeone/cloud-functions/*/config.json` → `routes[].src`)：完整 URL 正则（含 query string），`^/api-external/search/(.*)$` 通配一切
- **匹配优先级**：Edge Function 先于 Cloud Function → 如 Edge Function 模式匹配了 `/api-external/`，Cloud Function 永远收不到
