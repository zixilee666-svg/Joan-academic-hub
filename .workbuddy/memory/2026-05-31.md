# 2026-05-31 工作日志

## 01. 修复所有 /api/* 路由返回 HTML (commit: dcfae04)

### 问题
commit 82fc74a 将 `edge-functions/api/[[default]].js` 重命名为 `_kv-handler.js` 后，所有 `/api/*` 路由返回 `<!DOCTYPE html>` 而非 JSON，导致首页无法加载 spaces/papers 等数据。

### 根因分析
- `[[default]].js` 在 EdgeOne Pages 中是 catch-all 文件名，自动拦截所有 `/api/*`
- 重命名后，Edge Function 路由配置无法正确匹配，请求 fall-through 到静态页面
- 历次 regex 尝试失败原因：
  - `(?!ai/)` 负向先行断言：EdgeOne 正则引擎不支持
  - 字符类 `a[^i][a-z]*|[b-z][a-z]*`：Edge Function 文件系统 catch-all 仍覆盖 regex

### 修复方案
1. **非 catch-all 文件名**：`edge-functions/api/_api.js`（普通文件名）
2. **pages.config.json 双路由规则**：
   - `^/api/ai/conversations(/.*)?$` → `_api.js`（显式包含：对话需要 KV 存储）
   - `^/api/([^a][^/]*|a[^i][^/]*)(/.*)?$` → `_api.js`（排除 ai/）
3. **Cloud Function 接管**：`cloud-functions/api/ai/[[default]].js` → `/api/ai/chat` + `/api/ai/parse-paper`

### 路由验证结果
| URL | 处理器 |
|------|------|
| `/api/ai/conversations` | Edge Function (_api.js) ✓ |
| `/api/papers/123` | Edge Function ✓ |
| `/api/auth/login` | Edge Function ✓ |
| `/api/ai/chat` | **Cloud Function** ✓ |
| `/api/ai/parse-paper` | **Cloud Function** ✓ |
