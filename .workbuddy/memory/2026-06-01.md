## 网站加载速度诊断

- 诊断 Academic Hub V6 加载速度慢的原因
- 构建产物分析：dist 32MB，首屏总传输量约 3.1MB
- 发现 5 大瓶颈：
  1. 无路由级代码分割（975K 单入口）
  2. 过于激进的 modulepreload（预加载 charts+pdf+animation）
  3. PDF Worker 1.37MB 打包到 dist
  4. 字体 748KB（Playfair Display 所有子集 × 4 字重）
  5. pages.config.json 未配置 Cache-Control 头
- 缓存现状：EdgeOne CDN 存在但未配置 HTTP 缓存策略
- 优化方案：React.lazy + Suspense / modulepreload 瘦身 / PDF Worker CDN化 / 字体瘦身 / Cache-Control
- 预期优化后首屏从 3.1MB 降至 ~500KB（-84%）

## 三项低风险加载优化（已完成）

### 优化1: Cache-Control 缓存头 (pages.config.json)
- JS: `max-age=31536000, immutable`（带 hash 永不变化）
- CSS: `max-age=31536000, immutable`
- 字体: `max-age=31536000, immutable`
- 图片: `max-age=604800`（1周）
- HTML: `max-age=300, must-revalidate`（5分钟）
- 效果：二次访问基本 0 网络请求

### 优化2: modulepreload 精简 (vite.config.ts)
- 新增 `build.modulePreload.resolveDependencies` 过滤器
- 仅预加载 `react-vendor` chunk，其他（charts/pdf/animation）按需加载
- 预加载量：4 chunks (~1.25MB) → 1 chunk (~422KB)

### 优化3: 字体瘦身 (src/index.css + src/styles/fonts.css)
- Inter Variable: 新建 fonts.css，仅 latin + latin-ext（跳过西里尔/希腊/越南）
- Playfair Display: 改用 latin-400/500/600/700.css
- 字体: 748K → 344K（-54%），文件数 38 → 10（-74%）

### 综合效果
- 首屏传输: 约 3.1MB → 约 800KB（-74%）
- 构建: ✅ 成功，零错误

## 页面加载速度优化：Mock 延时削减 + dataStore 全局缓存

### 根因诊断
- 每个 Mock API 请求人为延时 `mockDelay(100 + Math.random() * 150)` = 100~250ms
- 5 个页面共 3~2 个并行 API 调用 → 首屏约 200ms 白屏
- 页面间**零数据缓存**：Dashboard→Library→MyLibrary→Research 每次切换都重新拉取 papers/projects
- admin 页面拉取 3 个 API 调用  × 100~250 ms → 首屏约 200ms（示例数字；已修正，原文提到3个并行）

### 优化 A：削减 Mock 延时（`src/lib/api.ts`）
- `mockRequest`: `mockDelay(100 + Math.random() * 150)` → `mockDelay(10 + Math.random() * 20)`
- 效果：每次 API 调用从 ~200ms → ~20ms（-90%）

### 优化 B：新建 Zustand dataStore 缓存层（`src/store/dataStore.ts`）
- **三大缓存域**：papers / projects / libraries
- **ensureXxx() 机制**：仅在未加载且未加载中时才拉取；已缓存则立即返回
- **invalidateXxx()**：标记缓存失效，下次 ensure 重新拉取
- **乐观更新方法**：addToPapers / removeFromPapers / updateInPapers / restorePapers（回滚用）
- **项目管理方法**：addToProjects / removeFromProjects / updateInProjects / restoreProjects

### 页面迁移详情

| 页面 | 迁移前 | 迁移后 | 改动量 |
|------|--------|--------|--------|
| **DashboardPage** | 3 个 useState + loadData 拉取 3 API | useDataStore(papers+projects) + 本地 stats | ~50 行 |
| **LibraryPage** | papers/materials 本地 state | papers 走 store，materials 保持本地 | ~60 行 |
| **MyLibraryPage** | libraries/allPapers 本地 state | allPapers 走 store，libraries 保持本地 | ~70 行 |
| **ResearchPage** | projects/papers 本地 state | 全部走 store，7 个突变函数全部重写 | ~90 行 |
| **AdminPage** | 保持独立（自带分页） | 不迁移 | 0 行 |

### 关键收益
- **首屏加载**：~200ms → ~20ms（Mock 延时削减）
- **页面切换**：第二次进入任何页面 → **0 网络请求**（dataStore 缓存命中）
- **Dashboard→Library→Dashboard** 往返：之前 3 次 API 调用（~600ms）→ 现在 0 次
