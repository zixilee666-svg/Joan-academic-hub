## AI Chat 三大问题修复 (00:29)

### 问题 1：504 Gateway Timeout（AI 对话超时）

**根因**：Cloud Function 平台默认超时 30 秒，但 AI API 调用设置了 55 秒。AI 响应慢时，Cloud Function 在 30 秒被平台强制终止 → 504。

**修复**：
- `cloud-functions/api/ai/[[default]].js`：`callOpenAICompatibleApi` 默认 timeout 55000→20000
- `handleAiChat` timeout 55000→20000
- `handleParsePaper` timeout 55000→20000
- `src/lib/api.ts`：`parsePaper` timeout 120000→25000

### 问题 2：AI 对话内容未保存

**根因**：Cloud Function 中 `handleSaveConversation`/`handleGetConversations` 是空实现（直接返回成功），无法访问 EdgeOne Pages KV Storage。

**修复**：`src/lib/api.ts` 中对话 CRUD 降级到 localStorage：
- `getConversations()` → localStorage `ai_conversations`
- `saveConversation()` → 写入 localStorage
- `deleteConversationRemote()` → 从 localStorage 删除

### 问题 3：论文 AI 分析模板无内容

**根因**：Cloud Function 的 system prompt 要求返回的字段与前端 `ParsedPaperResult` 结构不匹配：
- 后端返回 `journal`，前端期望 `venue`
- 后端返回 `number`，前端期望 `issue`
- 后端缺少 `month`、`references` 字段

**修复**：
- 重写 system prompt，明确要求返回 `venue`、`issue`、`month`、`references` 等字段
- 添加字段映射：`parsed.venue = parsed.venue || parsed.journal || ''`
- 添加字段映射：`parsed.issue = parsed.issue || parsed.number || ''`
- 将服务端生成的引用格式包装为 `citations: { bibtex, ieee, gb7714 }` 对象

## AI Chat SSE 流式传输修复 (00:43)

### 问题
部署后 AI 聊天仍然 504 Gateway Timeout，前端无回复显示。

### 根因分析
**Promise.race 超时只控制 HTTP 响应头到达时间，不控制响应体下载完成时间**：
1. DeepSeek API 响应头在 2 秒内到达 → `fetch` 赢得 `Promise.race`
2. 但完整 JSON 响应体（非流式，4096 tokens）需要额外 30+ 秒生成
3. Cloud Function 总执行时间超过平台 30 秒限制 → 504

### 修复 (commit 885ef66)

**方案：恢复 SSE 流式传输 + 前端解析器兼容 OpenAI 格式**

1. **Cloud Function** (`cloud-functions/api/ai/[[default]].js`):
   - `stream: false` → `stream: true`
   - `maxTokens: 4096` → `2048`（降低延迟）
   - 直接透传 `res.body`，无需等待完整响应

2. **前端** (`src/pages/AIChatPage.tsx`):
   - SSE 解析器兼容两种格式：
     - `{content: "..."}` (旧格式)
     - `{choices: [{delta: {content: "..."}}]}` (OpenAI/DeepSeek 格式)
   - 处理 `[DONE]` 结束标记

```javascript
const content = data.content || data.choices?.[0]?.delta?.content;
```

## PDF 文献解析修复 (07:50)

### 问题
用户上传 PDF 文件进行 AI 解析，返回 422 错误"AI 未能从文本中提取到有效的文献信息"。

### 根因
1. **PDF 文本提取质量差**：`fileExtractor.ts` 提取全篇 PDF（最多 50000 字符），包含大量正文噪音和编码问题（连字符、特殊字符）
2. **AI 无法从噪音文本中解析 JSON**：低质量文本导致 AI 返回非 JSON 格式内容
3. **无降级方案**：JSON 解析失败后直接返回 422，无正则提取兜底

### 修复 (commit 146d81a)

**前端** (`src/lib/fileExtractor.ts`):
- 只提取前 3 页（标题+作者+摘要），而非全篇
- 添加 PDF 元数据（Title/Author）作为文本头部
- 文本限制 15000 字符（原 50000）

**Cloud Function** (`cloud-functions/api/ai/[[default]].js`):
- 改进 system prompt：明确要求只输出 JSON，给出输出规则
- 降低 temperature：0.1 → 0.05（更确定性输出）
- 减少发送文本：4000 → 3000 字符
- 三层降级解析：`tryParseJson` → `extractWithRegex(AI内容)` → `extractWithRegex(原始文本)`
- `extractWithRegex`: 用正则从文本中提取 title/authors/year/abstract/venue/doi
- 错误日志增加 `parsedKeys` 便于调试

- `146d81a` pushed to origin/main ✅
- TypeScript 零错误 ✅
- Vite build 成功 ✅
