## AI Chat 三大问题修复 (00:29)

### 问题 1：504 Gateway Timeout（AI 对话超时）

**根因**：Cloud Function 平台默认超时 30 秒，但 AI API 调用设置了 55 秒。AI 响应慢时，Cloud Function 在 30 秒被平台强制终止 → 504。

**修复**：
- `cloud-functions/api/ai/[[default]].js`：`callOpenAICompatibleApi` 默认 timeout 55000→20000
- `handleAiChat` timeout 55000→20000
- `handleParsePaper` timeout 55000→20000
- `src/lib/api.ts`：`parsePaper` timeout 120000→25000

### 问题 2：AI 对话内容未保存

**根因**：Cloud Function 中 `handleSaveConversation`/`handleGetConversations` 是空实现（直接返回成功），无法访问 EdgeOne Pages KV Storage。

**修复**：`src/lib/api.ts` 中对话 CRUD 降级到 localStorage：
- `getConversations()` → localStorage `ai_conversations`
- `saveConversation()` → 写入 localStorage
- `deleteConversationRemote()` → 从 localStorage 删除

### 问题 3：论文 AI 分析模板无内容

**根因**：Cloud Function 的 system prompt 要求返回的字段与前端 `ParsedPaperResult` 结构不匹配：
- 后端返回 `journal`，前端期望 `venue`
- 后端返回 `number`，前端期望 `issue`
- 后端缺少 `month`、`references` 字段

**修复**：
- 重写 system prompt，明确要求返回 `venue`、`issue`、`month`、`references` 等字段
- 添加字段映射：`parsed.venue = parsed.venue || parsed.journal || ''`
- 添加字段映射：`parsed.issue = parsed.issue || parsed.number || ''`
- 将服务端生成的引用格式包装为 `citations: { bibtex, ieee, gb7714 }` 对象
