# 2026-06-02

## 修复 arXiv + Semantic Scholar 搜索 404 (commit: e102701)

**问题**: 导入导出页面搜索无结果，Network 显示 `No data found for resource with given identifier`，Console 报 `search_error: ApiError`

**根因**: 前端调用 `/api-external/search/arxiv` 和 `/api-external/search/semantic-scholar`，但 EdgeOne Pages 上该路径没有 edge function handler：
- `edge-functions/api/[[default]].js` 只处理 `/api/*` 路由
- `/api-external/*` 匹配 `pages.config.json` 的 `/* → ./dist` catch-all → 返回 `index.html`
- HTML 作为 JSON 解析失败 → 搜索报错

**修复 (3 层)**:
1. **前端路径修正**: `api.ts` 中 `searchArxiv` / `searchSemanticScholar` 路径从 `/api-external/search/*` → `/search/*`（`realRequest` 自动添加 `baseUrl=/api` → `/api/search/*`）
2. **Edge Function 路由**: 在 `segments[0] === 'search'` 处分发到 `handleSearchArxiv` / `handleSearchSemanticScholar`
3. **代理处理函数** (新增 ~180 行):
   - `handleSearchArxiv`: 代理 `export.arxiv.org/api/query`，解析 Atom XML → JSON
   - `handleSearchSemanticScholar`: 代理 `api.semanticscholar.org/graph/v1/paper/search`，透传 JSON
   - 辅助函数: `extractXmlTag`, `extractXmlAuthors`, `extractXmlYear`, `extractXmlLink`

**技术细节**:
- arXiv API 15s 超时，返回 Atom XML → 服务端解析为 JSON
- Semantic Scholar API 15s 超时，支持可选 `x-api-key` header
- ambos 使用 `User-Agent: JoanAcademicHub/1.0`
- TypeScript 零错误，Vite build 1.69s
